Technical Blog: Recursion

19th October 2014

Recursion is a function calling itself hopefully with the simpler versions of the original problem.

It consists of two main characteristics:
1.   An end goal, or base case (non recursive)
2.   Recursive case(s) : One or more cases that are different versions of the original problem

Example 1
Finding the factorial of a number using recursion


  def fact(n)
    return 1 if n==0
    return n*fact(n-1)
  end
  p fact(5) #120

The first line is defining a method factorial which takes n as a parameter.The second line is setting our base case, which is the point at which the recursion should stop happening. Any error writing this base case may take you to an infinite loop. For factorial, we want the recursion to go until we’ve gotten to 1.The third line is where we’re calling the method itself and setting up what’s actually happening: every time you call factorial method, you’re multiplying the given number by one less than itself. This continues till n becomes equal to 1 , which is when it executes the base case and returns the accumulated result.

The work flow of the solution is as follows:


fac(5)
    |
    5 * fact(4)
             |
             4 * fac(3)
                     |
                     3 * fact(2)
                              |
                              2 * fact(1)
                                       |
                                       1 * fact(0)
                                                |
                                                1
Output: 5 * 4 * 3 * 2 * 1 * 1 = 120

Example 2:
Finding whether a number is a fibonacii number or not


  def is_fib?(x=0, y=1, num)
    (num > y) && is_fib?(y,x+y,num) or num==y
  end

  p is_fib?(5) #true

Here it keeps calling the function is_fibonacci? till the number becomes less than or equal to y(=x+y), which is passed as a second parameter to the method. If y and the num are equal, it returns true else returns false

The work flow of the solution is as follows:


  is_fib?(x=0,y=1,5)
              |
        is_fib?(x=1,y=1,5)
                  |
                is_fib?(x=1,y=2,5)
                          |
                        is_fib?(x=2,y=3,5)
                                    |
                                is_fib(x=3,y=5,5)
                                          |
                                        num==y
                                        (5==5)
                                         true
